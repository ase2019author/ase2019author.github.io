Semistructured Merge in JavaScript Systems - Online Appendix

Abstract

The industry widely uses unstructured merge tools, that rely on textual analysis, to detect and resolve conflicts between parallel code contributions. On the other hand, semistructured merge tools go further by partially analyzing the syntactic structure and semantics of the code artifacts involved in a conflict. Previous studies compare these merge approaches, showing that semistructured merge is superior to unstructured one with respect to the number of reported conflicts, reducing the integration effort spent by developers, but, also, possibly negatively impacting the correctness of the merging process. However, these studies are based on semistructured merge tools that support different languages such as Java and C\#, but not JavaScript, the most popular programming language for the Web. In this paper, we implement different versions of semistructured tools that support JavaScript, and we conduct a study to compare them to an unstructured tool in order to better understand how semistructured merge works across different languages. We found evidences that semistructured merge approach for JavaScript reports fewer spurious conflicts than unstructured merge, without significantly impacting the integration correctness.

Keywords: Collaborative development, Software merging, Semistructured merge, Version control systems, JavaScript.
